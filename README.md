# Dynamic Model Selection Ensemble Learning for Energy Disaggregation in Agricultural Systems

This repository accompanies the manuscript *Dynamic Model Selection Ensemble Learning for Energy Disaggregation in Agricultural Systems* (Khaleghy, Clifford, Mason, 2026, submitted to PLOS ONE).

It contains the analysis notebook and a subset of the input data used in the paper.

## Repository layout

```
.
├── notebook.ipynb              Analysis notebook: training, evaluation, ensemble
├── data/
│   ├── pv-battery/             Scenario 3 inputs. SAM photovoltaic plus
│   │                           battery output for 13 simulated farms,
│   │                           hourly resolution, one CSV per farm
│   │                           (pv-bat1.csv ... pv-bat13.csv).
│   └── dairy-farm-pvs/         Scenario 2 inputs. SAM photovoltaic output
│                               for 13 simulated farms with heterogeneous
│                               PV capacities, one CSV per farm
│                               (PV1.csv ... PV14.csv).
├── LICENSE                     MIT for code, CC BY 4.0 for data.
└── README.md                   This file.
```

## What is included

- The analysis notebook, which trains GBRT, 1D CNN, CNN BiLSTM and Transformer base learners and produces the dynamic hour wise ensemble predictions and the leave one out ablation reported in the paper.
- The SAM derived photovoltaic time series for the 13 simulated farms under two scenarios: heterogeneous PV sizes (Scenario 2) and heterogeneous PV with battery storage (Scenario 3).

## What is not included

- The agent based dairy farm electricity consumption signal for the 13 farms. The signal was generated using the public simulator at <https://github.com/JLu2022/eire-aiai-dairy-energy-simulator> released under the MIT license. The notebook expects the consumption to be available as a separate input. Researchers who wish to fully reproduce the paper should generate this signal with the linked simulator.
- The Finland real world dataset used in Section 5.7 of the paper. Acknowledgement and access details for this dataset are provided in the manuscript.
- Trained model weights. These can be regenerated by running the notebook with the hyperparameters reported in Table 4 of the manuscript.

## How to use

1. Generate the dairy farm consumption signal using the linked simulator (see above).
2. Place the consumption file beside the data folders.
3. Open `notebook.ipynb` in JupyterLab or Kaggle, point the data paths to the files in `data/pv-battery/` and `data/dairy-farm-pvs/`, and run the notebook end to end.
